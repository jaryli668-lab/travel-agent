# 旅行规划Agent优化方案

> 文档版本: v1.0
> 创建日期: 2026-03-18
> 目标: 提升旅行规划结果的质量和用户满意度

---

## 一、问题分析

### 1.1 当前痛点

客户反馈：**规划结果不是最优路径**

### 1.2 根因分析

| 问题 | 原因 | 影响 |
|------|------|------|
| 路线绕路 | 缺少空间智能 | 用户体验差 |
| 价格不准 | 使用静态数据 | 预算超支 |
| 时间不合理 | 缺少时间估算 | 行程太赶或松散 |
| 信息过时 | 知识库非实时 | 遗漏重要信息 |

---

## 二、用户需求分析

### 2.1 旅行用户核心需求（按优先级）

```
1. 🛏️ 住宿体验 (40%)
   - 位置便利性（到景点/地铁的距离）
   - 性价比（价格 vs 质量）
   - 真实评价

2. 🗺️ 路线合理性 (30%)
   - 避免绕路（顺路游玩）
   - 时间分配合理
   - 节奏张弛有度

3. 💰 预算准确性 (20%)
   - 总预算可控
   - 隐性消费提示

4. 🎯 体验深度 (10%)
   - 当地特色/地道玩法
   - 避坑指南
```

### 3.2 数据源抽象设计

```python
# src/data_sources/base.py
from abc import ABC, abstractmethod
from typing import List, Dict, Any

class DataSource(ABC):
    """数据源抽象基类"""

    @abstractmethod
    async def fetch(self, query: str, context: Dict) -> List[Dict]:
        """获取数据并返回统一格式"""
        pass

    @abstractmethod
    def can_handle(self, query: str) -> bool:
        """判断是否能处理该请求"""
        pass

    @abstractmethod
    def get_priority(self) -> int:
        """返回优先级，数字越小优先级越高"""
        pass
```

## 四、核心优化模块

### 4.1 空间智能（地图服务集成）

**目标**: 解决路线绕路问题

**实现要点**:
- 集成高德/百度地图API
- 计算景点间距离和交通时间
- 根据距离自动规划"顺路"路线
- 推荐住宿位置（交通便利性评分）

**数据结构**:
```python
@dataclass
class Location:
    name: str
    address: str
    lng: float  # 经度
    lat: float  # 纬度

@dataclass
class RouteSegment:
    from: Location
    to: Location
    distance: int  # 米
    duration: int  # 分钟
    transport: str  # 交通方式
```

### 4.2 时间智能

**目标**: 合理分配游玩时间

**实现要点**:
- 每个景点推荐游玩时长（知识库+AI）
- 实时天气影响调整
- 开放时间检查
- 排队时间预估

**数据结构**:
```python
@dataclass
class TimeEstimate:
    visit_duration: int  # 游玩时长（分钟）
    travel_time: int  # 交通时间（分钟）
    queue_time: int  # 预估排队（分钟）
    best_visit_time: str  # 最佳游览时间
```

### 4.3 性价比优化

**目标**: 预算准确，最优价格

**实现要点**:
- 酒店实时比价
- 不同时期的预算预警
- 隐性消费提示
- 多平台价格聚合

### 4.4 多方案生成

**目标**: 给用户选择权

**方案类型**:
```
方案A：经济型（预算最优）
  - 优先选择性价比高的住宿
  - 更多公共交通
  - 免费景点为主

方案B：舒适型（体验优先）
  - 位置便利的住宿
  - 减少奔波
  - 深度体验

方案C：平衡型（推荐）
  - 性价比与体验平衡
  - 合理节奏
  - 预算可控
```

## 五、数据标注与训练

### 5.1 数据收集结构

```python
@dataclass
class InteractionLog:
    # 输入
    query_id: str
    user_input: str
    parsed_intent: Dict

    # 数据来源
    retrieved_knowledge: List[Dict]
    external_data: Dict  # 实时API数据
    map_data: Dict  # 地图服务数据

    # 输出
    llm_response: str
    generated_plan: Dict

    # 质量指标
    route_score: float  # 路线合理性评分
    budget_score: float  # 性价比评分
    time_score: float  # 时间利用率评分

    # 用户反馈
    user_rating: Optional[int]  # 1-5星
    user_feedback: Optional[str]

    # 元数据
    timestamp: datetime
    model_version: str
```

### 5.2 反馈收集机制

1. **即时反馈**: 每次行程规划后的评分按钮
2. **行程后调研**: 通过邮件/短信收集实际体验
3. **行为分析**: 用户修改了哪些部分

### 5.3 训练数据用途

- **Fine-tuning**: 用高质量对话数据微调模型
- **RAG优化**: 根据反馈调整知识库检索策略
- **Prompt优化**: 优化系统提示词
